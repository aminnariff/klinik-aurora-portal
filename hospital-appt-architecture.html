Opening

What we're building — and what's already done

5 min

We already have a working appointment system. This project extends it for a hospital context — the main additions are departments, doctor-owned scheduling, role-based staff access, and automated notifications. Core infrastructure (patient records, payments, WhatsApp, email) is already built and reusable.

Inform — no decision needed

V1 scope: Coordinator creates and manages all appointments on behalf of patients. Patients call or WhatsApp the clinic — no patient-facing self-booking in V1.

V2 (future): The system is designed so patients can eventually self-book via mobile app — same backend, no rework needed when the time comes.

Warm-up question

Walk me through a typical appointment today — from patient contact to appointment confirmed.

Helps catch any workflow assumptions we may have missed. No wrong answer.

Data Model

How the system is structured — Department is the new layer

10 min

The key structural change is inserting a Department layer between Branch and everything else. Each department owns its own doctors, services, and staff — this is what makes access isolation possible.

Decision needed

Are departments fixed units, or specialty-based?

Fixed departments (e.g. Orthopaedics, Cardiology) — each doctor belongs to one. Simpler. Specialty-based — a doctor has a specialty and can serve multiple departments. More flexible, more complex.

List every department that will be using the portal. Needed to set up initial data.

Scheduling

How doctor availability is managed

10 min

The coordinator manages each doctor's availability schedule. The system then presents open slots when booking. Patients can only be booked into times the doctor is marked as available.

How it works

Weekly schedule — Coordinator sets each doctor's regular hours (e.g. Dr. Ahmad: Mon–Wed 9am–5pm, 30-min slots).

Exceptions — Coordinator blocks off individual dates for leave, conferences, or holidays.

Live availability — When booking, system shows only open, unbooked slots for the selected doctor.

Decision needed

How long is each appointment slot — fixed duration, or different per service?

Simplest: every slot for a doctor is the same duration (e.g. 30 min) — easy to set up. Alternative: each service type has its own duration (consultation = 20 min, procedure = 60 min). Which fits better?

Their answer

Decision needed

Do doctors have the same schedule every week, or does it change often?

Consistent weekly schedule = set once, only update for exceptions. Frequently changing = coordinators need to adjust the schedule regularly.

Their answer

Decision needed

How far in advance can patients book — 2 weeks, 1 month, 3 months?

This controls how far out coordinators need to have schedules set up and ready.

Their answer

Patient Records

New patients — what information do you need at registration?

5 min

Patients are identified by their NRIC — each record is unique to the individual, no representative booking. When a new patient calls, the coordinator creates their profile before confirming the appointment.

Already in the system

Patient profile stores: NRIC, full name, phone number, email, date of birth, address, and medical card info. Need to confirm which fields are required to proceed with booking vs. optional to fill in later.

Decision needed

Queue Management

The live queue — how it looks and where it's displayed

5 min

Each doctor has a live queue for today's appointments. Status moves: Scheduled → Checked In → In Consultation → Done / No-Show. Coordinator or nurse marks check-in when the patient arrives.

What the queue shows

Now serving — patient currently in consultation
Next up — next checked-in patient
Upcoming — rest of today's schedule with times
Summary — how many waiting, how many done

Decision needed

Where is the queue displayed — wall screen, front-desk tablet, or doctor's own device?

Shared screens or multiple simultaneous devices need real-time WebSocket updates. A single coordinator's screen can use simple 30-second polling — much simpler to build.

Their answer

Decision needed

Who marks a patient as checked in — the coordinator, or the specialist?

In most clinics, the receptionist/coordinator marks arrival. If specialists also update status (e.g. mark "in consultation" when they start), they need access to do so.

Their answer
